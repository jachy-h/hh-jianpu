# 动态间距计算设计文档

## 1. 概述

动态间距计算旨在避免简谱渲染中的符号重叠问题，特别是八度点与相邻音符的重叠。系统基于符号宽度的静态预计算，使用二维边界框系统同时关注横向和纵向重叠问题。

## 2. 设计目标

- **避免符号重叠**：确保所有符号清晰可读，不会重叠
- **二维覆盖**：同时处理横向（相邻音符）和纵向（同一音符内）重叠
- **自适应间距**：根据符号复杂度动态调整间距
- **性能优化**：通过缓存边界框计算结果优化性能
- **智能默认值**：使用智能默认值，不需要配置选项

## 3. 架构设计

### 3.1 核心组件

1. **SymbolWidthCalculator**：计算每个符号类型的宽度（基于字体大小比例）
2. **BoundingBoxCalculator**：为每个音符计算二维边界框
3. **DynamicSpacingCalculator**：基于边界框计算自适应间距
4. **LayoutIntegrator**：集成到现有的layout.ts中，替换固定间距算法

### 3.2 数据流

```
Score AST → 音符边界框计算 → 自适应间距计算 → 音符位置确定 → ScoreLayout
```

## 4. 详细设计

### 4.1 符号宽度定义

基于字体大小比例定义符号宽度：

| 符号类型 | 宽度比例 | 说明 |
|---------|----------|------|
| 音符数字 | 100% | 一个字符宽度 |
| 八度点 | 30% | 点的大小 |
| 减时线 | 120% | 线的长度 |
| 波音标记 | 150% | 波浪的宽度 |
| 倚音 | 80% | 较小的字符 |
| 附点 | 30% | 点的大小 |

### 4.2 边界框计算

对于每个音符，计算二维边界框：

**横向边界框**：
- 最左边：倚音左边界
- 最右边：音符数字右边界 + 附点宽度

**纵向边界框**：
- 最上边：波音标记或八度点上边界
- 最下边：减时线或八度点下边界

边界框计算需要考虑所有可能的符号组合。

### 4.3 间距计算算法

自适应间距算法：

1. **横向间距**：
   - 对于每个音符，计算其边界框宽度
   - 对于相邻音符，确保它们的横向边界框不重叠
   - 间距 = 前一个音符的右边界 + 最小安全间距 + 后一个音符的左边界

2. **纵向间距**：
   - 对于每个音符，检查其纵向边界框内符号是否重叠
   - 如果重叠，调整符号位置（如八度点位置）

3. **最小安全间距**：10%的字体大小

### 4.4 集成策略

替换现有的固定间距算法：

1. 在`createLayout`函数中调用动态间距计算
2. 移除固定`noteSpacing`的使用
3. 基于计算出的间距确定音符位置

### 4.5 配置和默认值

使用智能默认值：
- 最小安全间距：10%的字体大小
- 边界框扩展：5%的额外空间
- 性能缓存：启用边界框缓存

## 5. 错误处理

### 5.1 边界框计算错误
- 回退到固定间距算法
- 记录错误日志

### 5.2 性能问题
- 如果计算时间过长，使用简化算法
- 监控计算性能

### 5.3 布局溢出
- 自动减少小节内音符数量
- 提供警告信息

## 6. 性能优化

### 6.1 缓存策略
- 缓存符号边界框计算结果
- 缓存音符边界框计算结果
- 使用Map进行快速查找

### 6.2 批量处理
- 批量处理音符，减少函数调用次数
- 预先计算所有符号宽度

## 7. 测试策略

### 7.1 单元测试
- 测试边界框计算准确性
- 测试间距算法正确性
- 测试错误处理逻辑

### 7.2 交互测试
- 使用现有测试用例验证
- 使用东北民谣-移调版验证实际效果
- 测试不同符号组合的渲染效果

## 8. 实现顺序

1. 实现符号宽度计算器
2. 实现边界框计算器
3. 实现动态间距计算器
4. 集成到layout.ts
5. 添加单元测试
6. 进行交互测试

## 9. 依赖关系

- 现有的`LayoutConfig`接口
- 现有的`NotePosition`和相关类型
- 字体大小配置

## 10. 风险评估

### 10.1 技术风险
- 边界框计算复杂度高
- 性能影响布局速度
- 与现有代码集成困难

### 10.2 缓解措施
- 使用缓存优化性能
- 逐步集成，保持向后兼容
- 充分的单元测试和交互测试

## 11. 验收标准

1. **功能验收**：
   - 所有符号清晰可读，无重叠
   - 八度点与减时线不重叠
   - 相邻音符不重叠

2. **性能验收**：
   - 布局计算时间增加不超过20%
   - 内存使用增加不超过30%

3. **兼容性验收**：
   - 现有测试用例全部通过
   - 东北民谣-移调版渲染正确
